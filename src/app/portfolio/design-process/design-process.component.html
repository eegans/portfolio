My Process

I can adapt my process to work within any project or company, and the scope and order of things is determined by external factors. Is the product already developed? Do you already have an existing design system in place? Do we have access to all the users for research and testing purposes? All of that can be discovered within the discovery or Understanding phase.

Different projects require different processes, and the process is often limited by budget and time. A perfect design process would only happen with zero limitations.

Understand

Understand the problem by holding meetings with stakeholders, defining objectives and requirements.

Research

User research by holding interviews, observation, task analysis.

Analyze

Organize and analyze your data! Compile all of your research into documents, create personas, workflows, and create user stories

Design

This is where you start creating deliverables. Sitemaps, wireframes, mockups, prototypes, Style Guides.

Test

Usability testing, getting user feedback via surveys.

Analyze Again

UX is never done! QA with development. Does this solve the user's problem? Perform additional usability tests. Continue to get feedback even after it has been deployed.
